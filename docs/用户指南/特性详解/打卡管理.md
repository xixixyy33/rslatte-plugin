# 打卡管理

## 概述

**打卡**侧栏视图类型为 **`rslatte-checkinpanel`**（`CheckinSidePanelView`）。用于在**当前空间**下：

- 多选 **打卡项**，查看 **月/年热力图**；
- **点击格子**快速 **打卡 / 取消打卡**（写日记、更新中央索引、可选 DB 同步、写入工作事件）；
- 用 **➕** 打开弹窗做 **指定日期的补录**（含备注）。

与 **「今日打卡」侧栏**（`rslatte-sidepanel`）不同：本侧栏专注 **打卡模块的完整能力**（热力图、索引维护），今日打卡侧栏是多种「今日」操作的聚合入口。

## 如何打开

- **命令**：`rslatte-open-checkin-panel`（**「打开侧边栏：打卡」**）

若 **打卡 Pipeline 模块**未启用，侧栏只显示「打卡模块未启用」。

---

## 标题栏按钮

| 图标 | 含义 |
|------|------|
| **➕** | **新增打卡**：打开 **`AddCheckinRecordModal`**。可选择 **任意日期**、打卡项、备注，确认后写入对应日期日记并走保存逻辑；若无可用打卡项，会提示先在 **设置** 中添加打卡项。 |
| **🧱** | **扫描重建打卡索引（全量）**。走 Pipeline `checkin` / `rebuild`；成功后 Notice「打卡索引已重建」，并尝试刷新今日侧栏数据、**按索引重算连续打卡天数**。 |
| **🗄** | **打卡归档**：将超过阈值天数的打卡 **索引记录** 归档（日记文件不移动）。 |
| **🔄** | **打卡手动刷新（增量）**：扫描阈值内变更日记；成功后 Notice「打卡已刷新」，同样会 **重算连续天数**（避免仅 normalize 导致 streak 不准）。 |

**🧱 / 🗄 / 🔄** 的显示受 **`uiHeaderButtons.checkin`** 控制（与财务等模块相同），可在设置中单独关闭；**➕** 始终显示。

---

## 期间选择

- **月份** / **年份** 切换与财务侧栏相同。  
- **月份**：`type="month"`，选 `YYYY-MM`。  
- **年份**：数字框，范围 **2000–2100**，非法或空会回退为当前年。

**切换空间**（插件内事件）会自动 **重新渲染** 本侧栏，热力图数据随当前空间变化。

---

## 选择打卡项

- 区块标题为 **「选择打卡项」**，左侧 **▶ / ▼** 可折叠整块。  
- 列表来自 **`recordRSLatte.getListsSnapshot(false)`** 中的 **`checkinItems`**，仅 **`active` 且未 `deletedAt`** 的项。  
- 名称后可跟难度角标：**🧠**（高脑力）、**🍃**（轻量）；「一般」难度无后缀（见 `formatCheckinDifficultyBadge`）。  
- **全选** / 单选勾选；**至少勾选一个** 才会渲染下方热力图，否则提示「请至少选择一个打卡项」。  
- **勾选状态保存在当前侧栏实例内存中**：关闭该侧栏视图再打开会 **清空勾选**，需重新选择（外部 URI/导航若调用 `applyNavFocus` 会自动勾选对应项）。

若无打卡项，显示「暂无打卡项，请先添加打卡项」（打卡项定义在 **RSLatte 设置** 的打卡相关配置中）。

## 打卡热力图

### 布局

- 标题 **「打卡热力图」**。  
- 每个已选且在该期间 **有数据** 的打卡项一块区域：`h6` 为打卡项名称，下方为网格。  
- 多块之间按 **打卡项名称** 排序。  
- 若所选期间内 **没有任何一条** 选中打卡项的记录，会提示 **「该月份/年份暂无选中打卡项的数据」**，此时不会出现分打卡项的热力图网格；**历史补录**可用标题栏 **➕**，保存后 **刷新索引** 再切回对应月份即可看到格子。

### 月视图

- 顶部一行 **日～六** 星期头。  
- 按日历对齐：月初前的格子为 **空白占位**，其后每天一格。  
- 格子样式：  
  - **已打卡**（当日有一条 **未删除** 记录）：`rslatte-stats-heatmap-checked`  
  - **未打卡**：`rslatte-stats-heatmap-unchecked`  
- 部分格子内显示 **日期数字**（如 1 日、每周第一天等，便于扫读）。  
- **悬停 `title`**：`YYYY-MM-DD 已打卡` 或 `… 未打卡`。

### 年视图

- 顶部同样一行星期头。  
- **每个月一行**：左侧 **「N月」**，右侧为该月日历网格，逻辑与月视图一致，仍按周对齐。

### 数据从哪来

- 优先 **`getCheckinStatsCache(currentSpaceId)`** 的 `items`；若无则回退为 **主索引 + 归档索引** 合并后的打卡记录。  
- 热力图只统计 **`isDelete` 为假** 的记录为「已打卡」。

### 点击格子（切换打卡）

- **已打卡** → 视为 **取消**：写入 **`is_delete: true`** 语义（与索引/日记一致）。  
- **未打卡** → **新增打卡**。  
- 流程摘要：  
  - 若日期等于 **今天**：调用 **`applyTodayCheckinRecord`** 更新今日侧栏等 UI。  
  - 若开启打卡 **DB 同步**：`upsertCheckinRecord`；失败时日志警告并 **本地仍继续**。  
  - **`recordRSLatte.upsertCheckinRecord`** 更新中央索引。  
  - **`appendJournalByModule("checkin", dateKey, line)`** 向当日日记追加一行（带 ✅/❌ 与时间）。日记失败会 Notice 并写审计日志。  
  - **`workEventSvc.append`**：`kind: checkin`，`action: create` 或 `delete`。  
  - **`refreshSidePanel`**（今日打卡等）+ 本侧栏 **`render`**。  
- 成功 Notice：**「已打卡」** 或 **「已取消打卡」**。

---

## 外部跳转与高亮

若其它模块调用 **`applyNavFocus({ recordDate, checkinId })`**：

- 强制 **月视图**，月份取 `recordDate` 的 `YYYY-MM`；  
- 将对应 **`checkinId` 加入勾选**；  
- 渲染后对 **`recordDate` 所在格子** 滚动可见并 **短暂高亮**（约 2.6s）。

---

## 与设置、索引的关系

- **打卡项定义、阈值归档、自动刷新** 等在 **RSLatte 设置 → 打卡** 相关区块配置。  
- 重建/刷新失败时 Notice 会带 Pipeline 返回的 **`message`**。  
- **`skipped`** 时可能无 Notice（引擎层门控，例如条件不满足时跳过）。
