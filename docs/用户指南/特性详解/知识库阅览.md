# 知识库阅览

## 概述

**知识库阅览**由 **`KnowledgeView`**（`src/ui/views/KnowledgeView.ts`）实现：在 Vault 内扫描 **`30-Knowledge`**（V2 知识库根，由 `resolveKnowledgeLibraryRootRel` 解析）下的 Markdown，提供 **推荐、统计概览、按目录分层清单**，并与 **输出索引** 在「概览」中串联展示。顶栏的 🔄/🧱 会驱动 Pipeline **`knowledge`** 模块刷新或重建 **`knowledge-index.json`**（供同步与其它逻辑使用）；**列表与推荐本身主要依赖 Vault 文件与 frontmatter**，索引文件读取多用于刷新后的条数提示等。

代码中注册 **两种视图类型**，**UI 相同**，便于在同一工作区 **并排** 打开两份（例如四象限布局右下角与工作区其它区域各一叶）：

| 视图类型常量 | ID | 构造参数 `host` | 典型场景 |
|----------------|-----|------------------|----------|
| `VIEW_TYPE_KNOWLEDGE` | `rslatte-knowledge` | `workspace`（默认） | Hub 工作流 **「知识」**、`activateKnowledgeView` 打开的右侧叶 |
| `VIEW_TYPE_KNOWLEDGE_PANEL` | `rslatte-knowledge-panel` | `sidepanel` | 手动将知识视图拖到侧栏、或布局里单独指定的侧栏叶 |

`getDisplayText()` 对两种 host 均返回 **「知识管理（工作台）」**；命令面板名称仍可能带「工作台」，与侧栏实例并存时不矛盾。

## 如何打开

- **命令**：`rslatte-open-publish-panel`（名称：**「打开侧边栏：知识管理（工作台）」**）— 通过 `createUiNavigation` 的 **`openKnowledgeWorkspace()`**：优先 **复用已有 `VIEW_TYPE_KNOWLEDGE` 叶**，否则在 **`getRightLeaf(false)`** 上 **`setViewState({ type: VIEW_TYPE_KNOWLEDGE })`**。  
- **Hub / 工作流**：**`activateWorkflowView("knowledge")`** → **`WORKFLOW_TO_VIEW_TYPE.knowledge`**（即 `rslatte-knowledge`）。  
- **URI**：`obsidian://rslatte-open?module=...`，下列键名等价打开本视图：`知识库`、`发布管理`、`知识管理（工作台）`、**`知识管理（阅览）`**、**`知识库阅览`**、`Knowledge`、`publish`、`knowledge`。  

**注意**：`activateKnowledgeView` / `activateKnowledgePanelView` / `activatePublishView` 当前 **均走同一打开逻辑**（工作台类型 `rslatte-knowledge`），**不会单独只打开 `rslatte-knowledge-panel`**；侧栏版需用户或 **四象限布局** 等自行放置 `VIEW_TYPE_KNOWLEDGE_PANEL`。

推荐布局 **四象限**（`quadrantWorkspaceLayout.ts`）中，**右下**页签组包含 **`VIEW_TYPE_TODAY`、`VIEW_TYPE_REVIEW`、`VIEW_TYPE_TIMELINE`、`VIEW_TYPE_KNOWLEDGE`**。

## 打开时的默认页签

**`onOpen`** 会将内部页签设为 **`random`（随便看看）** 并渲染。若存在待处理焦点路径 **`plugin.__rslatteKnowledgeFocusPath`**（见下节），**`render` 会强制切到「知识库清单」** 并展开对应一级/二级分组、滚动高亮该行。

## 三个页签

| 页签 ID | 界面文案 | 作用概要 |
|---------|-----------|----------|
| `random` | 随便看看 | 按评分算法 **每天固定推荐 3 篇**；点 **已看** 后仍留在当日名单，仅隐藏眼睛按钮；**次日**按阅读记录重算。行内可 **复制 output_id**、打开 **基础信息**。 |
| `overview` | 知识库概览 | 文档总数、近 30 天发布/阅读、从未阅读、「超 30 天且近 30 天未读」等；**最近 24 周**折线图（输出侧 **新增/完成** + 知识侧 **发布篇数** 与 **word_count 合计**）；**领域** 写作/发布洞察与 **领域标签云**；点领域后再点 **文档分类** 可列出 **知识 + 输出** 混合列表（点击知识行会 **聚焦知识清单**，点击输出行会 **打开输出侧栏并聚焦路径**，跨空间输出会提示切换空间）。 |
| `library` | 知识库清单 | 按 **一级目录（l1）/ 二级目录（l2）** 折叠树展示全部扫描到的 `.md`；**全部展开/全部折叠**；单条时间轴行支持 **预览打开**、**基础信息**、**复制 output_id**（「随便看看」另含 **已读** 按钮）。 |

**扫描上限**：单次收集最多 **`KNOWLEDGE_LIBRARY_SCAN_LIMIT`（5000）** 个 Markdown 文件。

### 目录层级（l1 / l2）如何算

在知识库根下递归收集文件后，相对路径会 **去掉最外层知识库文件夹名**（如 `30-Knowledge`），再取 **第一段为 l1、第二段为 l2**（不足则归为「未分一级/二级目录」）。排序默认倾向 **发布日优先**，无发布日则参考 **文件 mtime**（见 `collectKnowledgeMetas`）。

### 时间轴行与 frontmatter

- **发布日**：`published_at` / `publishedAt`，否则用文件 **mtime** 推本地日。  
- **最近阅读**：frontmatter 键 **`recent_read_dates`**（`KNOWLEDGE_REC_READ_KEY`），支持数组或分隔字符串，**最多保留 3 个日期**；点 **已看** 会写入当天并 **best-effort** `upsertKnowledgeIndexItemByFile`。  
- **列表圆点**：近 30 天内有发布或近 30 天内有阅读显示 **✅**，否则 **☐**。  
- **标签行展示**：`published_space_id` → 空间名、`output_document_kind` → 项目/一般、**领域**（`领域` / `domains` / `domain`）、**文档分类**（`文档分类` / `doc_category`）。  

点击行主体：**当前叶以预览模式打开该文件**（`openFile(..., { state: { mode: "preview" } })`）。

### 基础信息（📋）

**`KnowledgeDocInfoModal`**：汇总 **frontmatter**（优先展示一组预定义键的中英文标签）并与 **输出台账** 合并信息，便于核对 `output_id`、源输出路径、项目字段等（`src/ui/modals/KnowledgeDocInfoModal.ts`）。

## 顶栏操作

- **🔄**：**`runE2(..., "knowledge", "manual_refresh")`**，成功后读取索引并 Notice 条数。  
- **🧱**：**`runE2(..., "knowledge", "rebuild")`** — 全量扫描重建知识索引。  

**知识库 DB 同步**是否开启见 **`isKnowledgeDbSyncEnabled()`**（`knowledge-index.json` → `knowledge_docs` 等，需合法 API 与 **`knowledgePanel.enableDbSync`**），与阅览 UI 无硬绑定，但影响后端同步行为。

## 外部导航：`__rslatteKnowledgeFocusPath`

其它界面（例如 **概览** 中点击知识文档）可设置 **`plugin.__rslatteKnowledgeFocusPath`** 为 **vault 相对路径**，再 **reveal** 已有知识叶或 **`activateKnowledgeView`**。下一次 **`render`** 会：

1. 切换到 **知识库清单**；  
2. 展开包含该路径的 **l1/l2**；  
3. 对应行 **`rslatte-knowledge-focus-highlight`** 并 **`scrollIntoView`**；  
4. 清空该全局变量，避免重复触发。  

（输出侧类似变量为 **`__rslatteOutputFocusPath`**，由 **`gotoOutputPath`** 使用。）

## 与输出管理的关系

- **发布/完成输出** 的流程仍以 **输出侧栏** 为主；本视图 **不替代** 发布操作。  
- **概览** 周趋势与混合列表会读取 **`outputRSLatte.getSnapshot()`** 中的条目，与知识文档 **按领域/文档分类** 对照。  
- 输出侧代码亦会查找 **`VIEW_TYPE_KNOWLEDGE` / `VIEW_TYPE_KNOWLEDGE_PANEL`** 以配合跳转（与知识视图内 `gotoOutputPath` 互为补充）。

## 知识库物理路径与设置

- 根路径：**`getV2DirectoryPaths` → `knowledge`**（通常即 **`30-Knowledge`**），随 **`v2DirectoryRoot`** 等 V2 根配置变化。  
- **新建空间** 时会 **确保知识库目录树** 存在（`ensureKnowledgeLibraryTree`），与 **空间业务目录** 分离说明见 [空间与索引](空间与索引.md)。  
- **`knowledgePanel.secondarySubdirs`** 等用于 **发布/建目录** 等流程；阅览扫描的是 **整棵知识库根下** 的 md，不限于某一子配置。
