# 健康管理

## 概述

**健康**侧栏视图类型为 **`rslatte-healthpanel`**（`HealthSidePanelView`）。数据来自 **日记中的健康行** 与 **中央健康索引**（主索引 + 可选归档），并支持按月生成 **分析快照**（stats / alerts），用于睡眠、饮水、体重、饮食、周卡（腰围/血压/静息心率）等 **9 项合并指标** 的清单浏览与统计。

侧栏分两个主 Tab：**健康数据清单**（时间轴 + 筛选 + 可选趋势图）、**健康统计明细**（月快照汇总、衍生指标、滚动窗口、基础告警、规则告警）。

## 如何打开

- **命令**：`rslatte-open-health-panel`（**「打开侧边栏：健康」**）

若 **`isHealthModuleEnabled()`** 为假，侧栏仅提示在设置中开启 **「健康管理」**。

## 标题栏按钮

| 图标 | 含义 |
|------|------|
| **➕** | **新增健康记录**。打开 **`HealthCardModal`**（`AddHealthRecordModal.ts`）；保存成功后刷新侧栏。 |
| **🧱** | **扫描重建健康索引（全量）**。Pipeline `health` / `rebuild`；成功后 Notice「健康索引已重建」，并尝试 **`hydrateTodayFromRecordIndex`**。 |
| **🗄** | **健康索引归档**。超阈值记录从主索引迁入 **archive**（日记不移动）。 |
| **🔄** | **健康手动刷新（增量）**。Pipeline `manual_refresh`；成功后 Notice「健康已刷新」，同样会 hydrate 今日侧栏数据。 |

**🧱 / 🗄 / 🔄** 受 **`uiHeaderButtons.health`** 控制；**➕** 始终显示。

## 两个主 Tab

| Tab | 作用 |
|-----|------|
| **健康数据清单** | 主索引中的 **未删除** 记录，按日记日期分组时间轴；支持周期/数据项/日期筛选；筛选 **体重** 或 **腰围** 时可显示趋势图。 |
| **健康统计明细** | 选择自然月 **`YYYY-MM`**，读取 **月 stats / alerts 快照** 与 **分析索引**；支持刷新/回退快照（至多 3 版备份，与 Review/财务分析快照策略一致）。 |

## 「健康数据清单」页签

### 标题行

- 展示当前筛选结果 **条数**（`共 N 条`）。

### 筛选条（与财务清单风格一致）

| 控件 | 作用 |
|------|------|
| **数据周期** | **全部 / 日 / 周 / 月** — 对应索引项的 `period` 字段（展示角标 **【日】【周】【月】**，时间轴圆点 ○ / ◐ / ●）。 |
| **数据项** | 按 **9 项合并指标** 筛选：`体重、饮水量、睡眠、饮食、腰围、血压、静息心率、血糖、月经周期`（顺序见 `HEALTH_CANONICAL_METRICS_ORDER`）。选「全部」不过滤；`diet_level` / `diet_text` 等旧键会归并到 **饮食**。 |
| **日记日期** | `type="date"`，按 **`recordDate`** 精确筛选；**清除** 清空日期条件。 |

### 趋势图（条件显示）

- 当 **数据项** 选 **体重** 时：在列表上方绘制 **体重趋势**（`loadMergedHealthIndexItems` 合并主索引 + 归档，`collectDayWeightSeries`）；目标线来自设置中的 **目标体重**（`getHealthTargetWeightKg`）。  
- 当选择 **腰围** 时：绘制 **腰围周序列趋势**（`collectWeekWaistSeries`），目标来自 **`getHealthTargetWaistCm`**。

### 时间轴每一行

- **单击行**（非按钮）：在编辑器打开 **对应日记**，并滚动到该条 **主行**（优先 `sourceFilePath` + `sourceLineMain`，否则按 `entryId` / `metricKey` / `valueStr` 等在日记中反查）。  
- **左侧圆点**：日 ○、周 ◐、月 ●。  
- **标题**：周期角标 + 合并项短名 + 展示值（饮水会按设置 **`healthPanel.waterCupVolumeMl`** 换算展示；饮食支持热量档等）。  
- **副行 meta**：周卡/月卡引用（如 Wxx、Mxx）、饮食摘要、入睡时刻等。  
- **✏️**：打开 **`EditHealthEntryModal`** 修改本条；**无 `entry_id` 时禁用**（旧数据需重建索引补全 meta）。  
- **❌**：**撤销该条** — 向日记追加删除行 + meta，**`upsertHealthRecord` 标记删除**，并写 **WorkEvent**（`health` / `delete`）；需 **`entry_id`**。

空列表时：若无任何记录，提示用 **➕** 录入或 **重建索引**；若有记录但被筛掉，提示调整筛选。

### 外部跳转

**`applyLedgerNavFocus({ entryId, recordDate })`**：切到 **清单** Tab，并在 DOM 就绪后对匹配行 **高亮滚动**（与财务侧栏同构）。

## 「健康统计明细」页签

### 统计月份

- **`type="month"`** 选择器，默认 **当前月**。

### 快照操作条

| 按钮 | 作用 |
|------|------|
| **🔄 刷新当前年月分析快照** | 重算所选月的 **stats** 与 **规则告警**，写入 `health-analysis` 下月快照；写入前 **轮转备份**（至多保留 **3** 版）。 |
| **↩ 回退快照版本** | 将该月 **stats、alerts** 各 **回退一档**（依赖 `.bak1.json` 链）。 |

### 统计汇总（所选月）

- 若有 **月 stats 快照**：显示读取源、`periodKey`、`generatedAt`。  
- **KV 指标**（示例）：有效条数（月内）、**周期分布（日/周/月）**、睡眠 **不足 7h / 不足 5h** 天数、体重样本数与均值、规则告警计数（高/警/提示）。  
- 无快照时：提示先 **刷新/重建** 健康索引，或显示 **analysis index latest** 提示。

### 衍生指标（本月）

- 说明：**日卡按自然日去重**（同日多条取最新）；饮水达标对照设置 **`waterGoalCups`**（默认 8，与 `healthPanel` 配置一致）。  
- 可能包含：**睡眠**（有记录天数、日均时长、不足 7h/5h 天数、连续不足 7h 最长 streak）、**饮水**、**体重**（均值、高低、首尾差、相邻日最大波动）、**饮食**（高热量档天数与占比）、**周卡·腰围/血压/静息心率** 等 — 以当月快照内 `derived` 为准。  
- 无足够数据时显示灰字说明。

### 滚动窗口（近 7 日 / 近 30 日）

- **截止日**：快照 **anchor**（历史月用该月最后一天，当月用今天）；数据来自 **全索引日卡去重**。  
- 展示近 7、近 30 的睡眠/饮水/体重/饮食及 **日卡启用项全齐占比** 等（见 `statsSnap.rolling`）。

### 合并项末次值（月内最后一条）

- 按 **`HEALTH_CANONICAL_METRICS_ORDER`** 列出各合并项在 **该月最后一条** 的值、周期（日/周/月）、`recordDate`。

### 分析基础告警

- 依赖 **`health-analysis.alert-index.json`**。未生成时提示先 **刷新/重建**。  
- 否则展示状态、生成时间、缺失项列表（标题/详情/提示）。

### 规则告警（所选月）

- 读取 **`{YYYY-MM}.alerts.json`** 快照；列出 **未解决**（`status !== resolved`）条目，含严重度与文案。  
- 若带 **`relatedEntryIds`**：每条 id 旁有 **「打开 …」** 按钮，在索引中查到记录后 **`openHealthRecordInDiary`**；找不到则 Notice 建议 **重建索引**。

## 设置相关

- **`healthPanel.waterCupVolumeMl`**：单杯毫升数（默认 500，限制 50–2000），影响饮水展示与日记反查。  
- **`healthPanel.waterGoalCups`**：饮水达标杯数（1–30，默认 8），用于衍生指标与滚动窗口达标率。  
- **目标体重 / 目标腰围**：用于清单页趋势图参考线（见 `getHealthTargetWeightKg` / `getHealthTargetWaistCm`）。
