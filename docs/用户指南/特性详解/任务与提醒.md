# 任务与提醒

## 概述

**任务管理**侧栏由 **`TaskSidePanelView`** 实现，主视图类型为 **`rslatte-taskpanel`**。同一套 UI 内用 **三个子页签** 切换：

| 子页签 | 界面标题 | Pipeline 模块 | 典型索引文件（当前空间 `index/` 下） |
|--------|----------|---------------|----------------------------------------|
| **事项提醒** | ⏰ 事项提醒 | `memo` | `memo-index.json` |
| **日程安排** | 📅 日程安排 | `schedule` | `schedule-index.json` |
| **任务清单** | 🗂 任务清单 | `task` | `task-index.json` |

条目来源为 Vault 内 **日记/任务文件夹** 中的 Markdown 行，经扫描解析后写入索引；侧栏只做 **展示、筛选、跳转与写回**（新增/编辑/状态变更等通过各类 Modal 与 **`taskWriteFacade` / `memoWriteFacade` / `scheduleWriteFacade`** 等）。

插件还注册了 **`rslatte-schedulepanel`**，与上者 **同一类** 的第二个视图类型（兼容旧布局）；**`getViewType()`** 仍返回 **`rslatte-taskpanel`**。日常以命令打开的 **`rslatte-taskpanel`** 叶为准，在叶内切换 **日程安排** 即可。

> Hub 顶栏工作流 **未**单独放「任务管理」，一般通过 **命令** 或其它视图中的链接打开。

## 如何打开

- **命令**：`rslatte-open-taskpanel`（**「打开侧边栏：任务」**）  
- **URI**：`obsidian://rslatte-open?module=任务管理`（及 `main.ts` 中 `moduleMap` 里等价别名，如 `task` 等）  
- **程序化**：**`activateTaskView(opts?)`**，可选 **`{ subTab: "memo" | "schedule" | "task" }`**，在叶显示后 **`switchToSubTab`** 落到对应页签。

## 模块开关与空状态

子页签是否出现由 **`moduleEnabledV2`** 决定（侧栏内与 Pipeline 门控逻辑一致，见 **`TaskSidePanelView.render`**）：

- **`task`**：任务清单整页。  
- **`memo`**：事项提醒。  
- **`schedule`**：日程安排；若未单独配置布尔值，**默认跟随 `memo`**（与代码中 `scheduleEnabled = normalizeBool(me2.schedule, memoEnabled)` 一致）。  

若 **三者皆关**，侧栏 **不渲染内容**（无「模块已关闭」占位，与联系人侧栏策略不同）。

每个子分区顶栏另有 **DB 同步状态灯**（`task` / `memo` / `schedule` 各自一条），依赖设置中的 **API 与模块 DB 同步开关**。

## 事项提醒（memo）

- **➕**：**`AddMemoModal`** — 新增提醒（写入 **今日日记** 约定位置）。  
- **🧱 / 🗄 / 🔄**（若 UI 配置显示）：分别对应 Pipeline **`rebuild`**、**`manual_archive`**、**`manual_refresh`**。  
- **四分组**（可折叠；「活跃 / 近期闭环」默认收起）：  
  - **今日关注**  
  - **已超期**  
  - **活跃条目**  
  - **近期完成/取消/失效（N 天）** — 窗口由 **`recentClosedMemoWindowDays`** 等与 `taskPanel` 相关设置约束。  
- **即将到期**等范围与 **`reminderUpcomingDays`** 相关；加载有 **超时保护**（过长索引/大量文件可能触发提示）。  
- 展示用的 **`memo_tags` 业务日** 可与 **`getMemoIndexTagsDerivedDay`** 对齐（与 **日历日** 可能不同，见索引注释）。

## 日程安排（schedule）

- 数据来自对日记等文件中 **符合日程语法的行** 的扫描，由 **schedule Pipeline** 分桶写入 **`schedule-index.json`**（与「写在 memo 行里再识别为日程」的管线一致，见源码注释）。  
- **➕**：**`AddScheduleModal`**。  
- **🧱 / 🗄 / 🔄**：对 **`schedule`** 模块的 rebuild / archive / manual_refresh。  
- **分组**：**今日安排**、**近期日程安排**、**已超期**、**活跃条目**、**近期完成/取消/失效**（天数与 **`scheduleRecentClosedDays`** 等设置相关）。  

与 **日程日历**侧栏（**`rslatte-calendar`**，月历 + 选日展开）互补：本页为 **列表/时间轴** 视角，日历为 **月视图**（见 [日程日历](日程日历.md)）。

## 任务清单（task）

- **➕**：**`AddTaskModal`** — 新增任务（写入 **今日日记**）。  
- **🧱 / 🗄 / 🔄**：对 **`task`** 模块的 rebuild / manual_archive / manual_refresh。  
- **固定分区**（数据来自 **`getTaskListsForSidePanel`**）：  
  1. **⭐ 重点关注** — Top N 由 **`focusTopN`** 控制（约 3～10），不可折叠。  
  2. **今日处理清单** — 子区：**今日行动**、**今日跟进**、**超期/即将超期**、**其他风险**；标题行提供 **全部展开/折叠**。  
  3. **其他活跃清单** — 默认 **收起**。  
  4. **近期闭环** — 子区：**近期取消**、**近期完成**；亦可一键展开/折叠。  

任务卡片支持 **标签芯片**（**`task_tags` / `TASK_TAG_META`**）、**关联日程行**、点击打开 **源文件与行号** 等。列表折叠状态部分持久化在 **`taskPanel.collapsedLists`**。

**「今日」与标签推导**使用 **`getTaskTodayKey(taskPanel)`**，与 **Today、Capture、Inbox 统计** 等口径一致。

## 刷新、重建与自动刷新

- **手动 🔄**：仅刷新 **当前子页签对应模块** 的索引流程（task / memo / schedule 各自一条 `manual_refresh`）。  
- **联系人侧栏** 的全局 🔄 会 **串联** contacts + task + memo + schedule + project，与本侧栏单按钮不同。  
- **重建 🧱**：对应模块 **全量扫描重建** 索引，耗时长，仅在异常或迁移后使用。  
- **归档 🗄**：将 **已闭环** 条目的索引迁入 **archive 分片**（笔记正文仍在日记中，与按钮 title 说明一致）。  
- **自动刷新**：由插件 **`autoRefreshIndexEnabled` / `autoRefreshIndexIntervalMin`**（分钟）统一驱动（侧栏内不再单独跑 timer，见 **`TaskSidePanelView.onOpen`** 注释）。

## 外部跳转与定位

- **`switchToSubTab(tab)`**：仅在对应模块开启时生效。  
- **`scrollToTask(filePath, lineNo)`**：**`lineNo` 为 0-based**，用于从联系人等跳转到 **任务清单** 中的行（联系人索引里为 1-based，调用方会换算）。  
- **`focusTaskRowByFileLine` / `focusMemoRowByFileLine`**：先切页签再展开分区并滚动高亮；失败则 **打开源笔记**。  

## 与其它功能的关系

- **今天（Today）**：今日执行清单、统计等大量依赖 **`getTaskListsForSidePanel`**、日程/提醒分桶，与任务侧栏 **同一套索引**。  
- **快速记录**：「今日任务」写入日记后进入 **task** 索引体系。  
- **项目管理**：项目任务在 **项目侧栏** 为主；任务侧栏内曾有「里程碑任务」区块的实现，**当前未挂入渲染**（保留代码）。  
- **联系人**：任务/提醒描述中的 **`[[C_uid|…]]`** 等会进入联系人 **互动索引**。
