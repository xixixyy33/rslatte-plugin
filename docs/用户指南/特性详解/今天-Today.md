# 今天（Today）

## 概述

**今天**侧栏视图类型为 **`rslatte-today`**（`TodayView`），与 **回顾** 类似采用 **三个子页签**：**今日执行**、**今日核对**、**今日记录**。侧重把「今天要做什么」与「今天发生了什么」放在同一侧栏里：执行页聚合任务/日程/项目推进；核对页按天汇总闭环与变更；记录页用打卡/财务/健康/日记的摘要灯与卡片跳转到底层模块。

实现上，执行统计与清单依赖 **`todayExecuteStats`** / 任务与日程索引；记录页由 **`todayRecordsModel`** + **`todayRecordsRender`** 渲染；核对页由 **`todayReconcileZonesModel`** + **`recordReconcileRender`** 异步构建模型后渲染。

## 如何打开

- **命令**：`rslatte-open-today-view`（**「打开侧边栏：今天」**）→ 内部等价于 **`activateWorkflowView("today")`**，打开或聚焦 **`VIEW_TYPE_TODAY`** 叶。
- **Hub**：工作流条 **「今天」**（`WorkflowViewId` 为 `today`）。

### 子页签与 Obsidian 标签标题

- 子页签文案：**今日执行**、**今日核对**、**今日记录**。
- **`getDisplayText()`**：在 **今日核对** 时显示「今日核对」，在 **今日记录** 时显示「今日记录」，否则显示 **「今天」**（与默认子页签「今日执行」对应）。

### 首帧落在「今日核对」（扩展）

源码导出 **`markPendingOpenTodayReconcileTab()`**：置位后，**下一次** `TodayView.onOpen` 会把 **`_subTab`** 设为 **`reconcile`** 并消费该标志。已打开 Today 时可用实例方法 **`openReconcileSubTab()`** 切换到核对子页签并 **`render()`**（同时清除待打开标志）。便于工作流或外链在打开侧栏时直接进入核对视图。

## 顶栏与刷新（今日执行 / 通用）

在 **今日执行** 子页签中，顶栏展示标题 **今日执行**、当前 **任务「今日」键**（**`getTaskTodayKey(taskPanel)`** 的 **`todayKey`**，与任务索引按日打标签的维度一致；极端情况下可能与纯日历日不完全相同），以及 **🔄 刷新**（调用 **`refresh()`** → **`render()`**），用于重拉清单与统计。

## 今日执行

分为两大区域：**执行清单**、**执行统计**。

### 执行清单

按界面顺序包含：

| 小节 | 说明 |
|------|------|
| **🔥 今日重点处理** | 聚焦任务（如置顶/星标等规则）+ 带「今日应处理」类标签的星标项目任务 + 「今日关注」提醒等 |
| **📋 今日行动清单** | 当日行动向任务列表 |
| **🚀 项目推进项** | 项目 next action / 推进相关卡片 |
| **⏳ 等待中 / 跟进中事项** | 等待、跟进状态条目 |
| **🗓 今日日程** | 当日日程条目 |
| **⏰ 超期 / 风险任务** | 命中超期、高拖延、假活跃等标签的任务卡片 |

卡片交互以 **`activateTaskView` / `activateProjectView`** 等为主，并尽量 **`focusTaskRow` / `scrollToProject`** 定位到具体行或项目。

数据侧会拉取任务侧栏列表、日程/提醒分桶，并对 **`projectMgr.ensureReady`**（含快照 hydrate）以保证项目相关展示一致。

### 执行统计

区域标题为 **执行统计**；标题行提供 **「工作台」** 链接 → **`activateHubView`**。

以下为可点击统计块（文案与实现 **`buildTodayExecuteStatsModel`** / `TodayView` 内渲染一致）：

- **📥 Inbox / 快速记录**：今日新增与未清理 backlog → **`openStatsCaptureInbox`**
- **🎯 活跃项目**：活跃 / 推进中 / 风险 / 假活跃 → **`openStatsProjectList`**
- **🗂 任务风险**：已超期 / 高拖延 / 假活跃 → **`openStatsTodayHandling`**
- **⏸ 等待推进**：等待与跟进的总量及「今日」子计数、高风险拖延 → **`openStatsTodayHandling`**
- **👤 联系人动态**：生日、待跟进、未互动等子行（可展开人名）
- **✍ 输出候选**：一般 / 项目 → **`openStatsOutputInProgress`**

## 今日核对

由 **`renderRecordReconcileBody`** 渲染；数据来自 **`buildTodayReconcileZonesModel`**。源码注释称「七分区」，界面 **一级大区标题** 实为下列 **六项**（其下仍有嵌套子块）：

1. **今日跟进** — **🧭 跟进一览**：任务 / 联系人 / 项目 进度类状态点（🟢/⚪），点击跳转对应侧栏（任务、联系人、项目等）。
2. **今日新增** — **📋 分时段新增**：按凌晨、上午、下午、晚上汇总当日新建计数（任务、提醒、日程、快速记录、项目任务、联系人、输出等），可点类型跳转对应模块。
3. **今日闭环**：**今日完成项**（任务、项目任务）、**今日关闭项**（日程、提醒）、**今日输出结果**（完成整理的草稿、发布）等分组卡片。
4. **今日日程轨迹（已完成）** — **🛤 已完成泳道**：当日已完成日程的时间轴泳道可视化。
5. **今日更新**：任务状态变化、延期/改期、等待/跟进变化、项目推进变化（next action）、联系人变化（今日互动卡片、资料/跟进字段更新）等。
6. **今日复盘摘要**：任务 / 项目 / 日程 / 联系人 / 输出 等摘要卡片，点击跳转对应侧栏。

## 今日记录

由 **`renderTodayRecordsBody`** 绘制正文；模型 **`buildTodayRecordsModel`**。分区版式与「今日执行」对齐（大区标题 + `h4` 小节）。模块未开启时对应整块不出现。

- **今日记录状态**：**📊 今日记录状态** — 打卡 / 财务 / 日记 / 健康 四盏灯（🟢/⚪），点击打开 **今日打卡侧栏**（`activateRSLatteView`）并 **`inspectSection`** 滚到对应分区。
- **打卡**（若启用）：**✅ 今日打卡状态**、**📇 今日打卡记录**（卡片可跳转打卡管理并定位日期格）。
- **财务**（若启用）：**💰 今日财务记录状态**、**📒 今日财务记录**、**⚠️ 今日财务告警**。
- **健康**（若启用）：**❤️ 今日健康状态**、**📇 今日健康记录**、**⚠️ 今日健康告警**（告警区当前有与 health-analysis 快照接入相关的占位说明）。
- **日记**（若启用）：**📝 今日日记记录状态**（按配置的子窗口展示是否有内容）。
- **今日复盘摘要**：汇总卡片如打卡完成进度条、财务笔数与净额、健康日维完成度、日记总字数与子窗口条形对比等，点击跳转各管理侧栏或统计 Tab。

## 与「今日打卡」侧栏、工作台的区别

| 对比 | **今天（Today）** | **今日打卡侧栏**（`rslatte-sidepanel`） | **工作台（Hub）** |
|------|-------------------|------------------------------------------|-------------------|
| 核心用途 | 执行 + 核对 + 跨模块今日摘要 | 当日打卡/记账/健康/日记的 **操作台** | 入口导航与空间级总览 |
| 视图类型 | `rslatte-today` | `rslatte-sidepanel` | `rslatte-hub` |
| 典型命令 | `rslatte-open-today-view` | `rslatte-open-sidepanel` | Hub 专用打开方式 |

命令面板里的 **「今天」** 指本视图；**「打开侧边栏：今日打卡」** 是另一类侧栏，不要混用。今日记录页大量 **跳转** 到今日打卡侧栏或财务/健康/打卡管理，而非在本页完成录入。

## 易混淆名称

- **「今天」**（命令 / Hub）→ 本 **`TodayView`**。  
- **「今日打卡」** → **`RSLatteSidePanelView`**（见 [今日打卡侧栏](今日打卡侧栏.md)）。
