# 联系人管理

## 概述

**联系人管理**侧栏用于浏览、搜索、编辑联系人笔记，并查看 **跨笔记聚合的互动记录**（任务、提醒、日程、项目任务、手动记互动等）。联系人数据以 Vault 内 **Markdown + YAML frontmatter** 为主；**主索引**与 **互动索引** 落在中央索引目录（按空间分桶），供侧栏快速加载与跳转。

- 视图类型：**`rslatte-contactspanel`**  
- 实现类：**`ContactsSidePanelView`**（`src/ui/views/ContactsSidePanelView.ts`）  
- 索引读写：**`ContactsIndexStore`** / **`ContactsIndexService`**（`src/contactsRSLatte/`）

**模块开关**：需 **`moduleEnabledV2.contacts === true`**，且通过 **插件初始化环境检查** 门控后，侧栏才展示完整功能；否则仅显示「联系人模块未启用」（视图仍会注册，便于后续开启模块）。

## 如何打开

- **命令**：`rslatte-open-contacts-panel`（**「打开侧边栏：联系人」**）  
- **URI**（`main.ts` 中 `moduleMap`）：`obsidian://rslatte-open?module=联系人` 或 `module=contacts`  

**外部定位**：**`focusContactByUid(uid)`** 会选中对应名片并刷新（例如 **Today**、**今日核对** 等处的联系人跳转）。

> Hub 顶栏工作流条 **未**单独列出「联系人」，一般通过命令、URI 或其它侧栏链接打开。

## Vault 目录与分组

- **`contactsModule.contactsDir`**：联系人根目录（默认 **`90-Contacts`**）。  
- **`contactsModule.archiveDir`**：归档目录（默认 **`{contactsDir}/_archived`**）。  

**分组（group）**：侧栏下拉中的分组名来自根目录下 **一级子文件夹**（排除 **`.attachments`**、**`.rslatte`**、以及位于根下的 **归档文件夹** 名）。联系人笔记路径需与索引中的 **`group_name`** 一致才能被筛到对应分组。

联系人文件内通过 frontmatter 维护 **`contact_uid`**、**`display_name`**、**`status`**（如 `active` / `cancelled`）等字段；取消联系人时会更新 frontmatter，并可按配置 **失效关联的生日提醒**（`extra.birth_memo_uids` 等），详见源码 **`setCancelled`**。

## 中央索引（按空间）

在 **`resolveSpaceIndexDir(settings, spaceId)`** 下常见文件：

| 文件 | 作用 |
|------|------|
| **`contacts-index.json`** | 主索引：联系人条目列表、解析错误文件记录等 |
| **`contacts-interactions.json`** | 互动索引：按联系人、按源文件聚合的互动条目 |
| **`archive/contacts-archive-index.json`** | 已归档联系人主索引镜像 |

重建/刷新由 Pipeline（如 **`contacts`** 模块的 **`rebuild`** / **`manual_refresh`**）驱动；与旧版「无 space 分桶」路径的兼容读取见 **`indexStore`** 注释。

## 侧栏布局与操作

### 顶栏

- 标题 **🪪 联系人** + **DB 同步状态灯**（与其它模块一致，依赖 **`contactsModule.enableDbSync`** 与合法 **`apiBaseUrl`**）。  
- **➕ 新增联系人**：`AddContactModal`，成功后 **`rebuildContactsIndex`** 并刷新。  
- **🧱 重建**（若 UI 配置显示）：仅扫描 **联系人目录（不含归档树）** 重建主索引；归档索引由 **笔记归档** 流程维护。  
- **🗄 笔记归档**（若显示）：将 **已取消且满足阈值** 的联系人 md 移入归档目录，并刷新主索引与 **`contacts-archive-index`**。  
- **🔄 刷新**：依次 **`contacts` → `task` → `memo` → `schedule` → `project`** 的 **`manual_refresh`**，再 **`refreshAllContactNoteDynamicBlocks`**，保证互动索引与笔记内 **「动态互动」生成块** 一致。

### 列表区

- **搜索**：对 **姓名、aliases、tags** 子串匹配（**Enter / 🔍** 提交；输入框用 draft 避免刷新打断 **IME**）。  
- **分组下拉**：`All` 或具体文件夹名。  
- **状态**：`active`（默认，不含 cancelled）/ `cancelled` / `all`。  
- 列表按 **sortname / 显示名** 分组排序（时间轴样式 + 首字母桶）。

点击一行进入右侧 **名片**；再次刷新会保持选中 uid（若仍存在）。

### 名片

- **点击头像/标题区**：在编辑器中打开联系人源文件（若已打开则聚焦该页签）。  
- **统计**（若可算）：**今日新增互动**、**最后互动**、**超期**（与 **`contactFollowupOverdueDays`**、任务面板的「今日」键等一致，见 **`buildInteractionStats`**）。  
- **📝 记互动**：`AddContactManualEventModal`，写入联系人笔记 **静态互动区**（与动态块分离）。  
- **✏️ 编辑**：`EditContactModal`。  
- **⛔ / ✅**：将 frontmatter **`status`** 设为 **`cancelled`** 或恢复 **`active`**，并 best-effort **DB 同步**、**WorkEvent**、重建索引。

#### 子页签：互动记录 / 联系人详细信息

- **互动记录**：从 **`contacts-interactions.json`** 查询当前 uid；可筛选 **来源类型**（全部 / 任务 / 提醒 / 日程 / 项目任务 / 手动）、**仅未完成**、**最近 N 条**（10/20/50）。条目可点击或 **↗** 跳转源位置：  
  - **`task`**：打开任务侧栏并 **`scrollToTask`**  
  - **`project_task`**：打开项目侧栏并 **`scrollToProject`**（含里程碑解析）  
  - 其它：优先 **`openNoteAtLine`** 类导航打开文件并定位行号（索引 **`line_no`** 为 **1-based**，与任务列表 DOM 的 0-based 在代码中已换算）  
- **联系人详细信息**：展示 frontmatter 中 **不在名片固定字段且未被黑名单屏蔽** 的键值（**`contactDetailsFieldBlacklist`**）。

渲染时会 **异步** 调用 **`replaceContactDynamicGeneratedBlock`**，把摘要写入联系人笔记中受控的 **「动态互动」** 块（失败不影响侧栏；与 **`eventSectionHeader`**、**`dynamicEventSubHeader`** 等设置一致）。

## 编辑器与引用

### 命令：插入联系人信息

- **`rslatte-contacts-insert-reference`**（**「插入联系人信息」**）  
- 优先当前 **Markdown 编辑**；若无焦点则从任意已打开的 Markdown 页注入。  
- 弹出选择器后插入 **Wiki 链语法**：**`[[C_<uid>|显示名]]`**（见 **`InsertContactReferenceModal`**）。  
- 索引为空时会提示先 **重建联系人索引**。

任务/提醒/日程/项目任务等 **多个模态** 内也会调用 **`openContactReferencePicker`**，在描述中插入同一格式链接。

### 阅读 / 预览模式

插件对预览中的联系人链接有 **后处理**（如悬停卡片，`setupContactsLinkPostProcessor`），具体表现以当前版本 UI 为准。

侧栏与部分列表使用 **`renderTextWithContactRefs`**：可将正文中的联系人链解析为可点击样式，并在需要时 **打开联系人侧栏**。

## 与其它模块的联动

- **任务 / 提醒 / 日程**：描述或 meta 中出现 **`[[C_uid|…]]`** 或跟进字段时，可被扫入 **互动索引**。  
- **项目任务**：`项目任务清单.md` 内引用会进入 **`project_task`** 来源；项目状态变更后可 **`refreshContactInteractionsForTasklistFile`**。  
- **日记内任务**：新建任务后可 **`refreshContactInteractionsForTaskFile`** 更新互动索引与笔记动态块。  

操作日志（WorkEvent）中 **`kind: contact`** 记录侧栏取消/恢复、记互动等相关事件（见 **`workEventService`** 注册表）。

## 设置项摘要（节选）

| 配置 | 含义 |
|------|------|
| **`contactsDir` / `archiveDir`** | Vault 内联系人根目录与归档目录 |
| **`enableDbSync`** | 是否与后端同步（另需合法 API 地址） |
| **`contactFollowupOverdueDays`** | 「超期」文案阈值（天） |
| **`interactionTimelinePreviewCount`** | 每条互动下展示 **`interaction_events`** 预览条数 |
| **`contactDetailsFieldBlacklist`** | 「详细信息」页签隐藏的 frontmatter 键 |
| **`eventSectionHeader` / `dynamicEventSubHeader`** | 笔记内静态互动标题、动态生成块子标题 |
| **`contactsInteractionArchiveShardMaxBytes`** 等 | 互动索引分片/归档相关（见设置页与源码） |

**`contactsModule` 为空间级设置**（见 [空间与索引](空间与索引.md)），换空间后目录与开关可能不同。

## 与其他入口的区别

| 入口 | 差异 |
|------|------|
| **任务管理** | 以任务/提醒/日程列表为主；联系人侧栏以 **人为中心** 聚合多条来源。 |
| **项目管理** | 以项目文件夹与里程碑为主；联系人侧栏仅从 **项目任务行** 回跳。 |
| **Today** | 展示今日执行与统计；可跳转联系人但不维护联系人主数据。 |
