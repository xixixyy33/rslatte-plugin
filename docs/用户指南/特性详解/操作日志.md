# 操作日志

## 概述

**操作日志**侧栏由原「时间轴」视图演化而来，视图类型 **`rslatte-stats-timeline`**，实现类 `TimelineView`（`src/ui/views/stats/TimelineView.ts`）。它按**时间**列出 **工作事件（Work Event）**：在 RSLatte 各模块中的创建、更新、状态变更、发布等行为，写入库内 **JSONL** 日志后，由本视图读取、筛选并以**时间轴**展示。**点击单条事件**可尝试打开对应的业务侧栏（见下文「跳转规则」）。

## 如何打开

- **命令**：`rslatte-open-timeline`（命令面板：**「打开侧边栏：操作日志」**）
- **Hub**：工作流 **「操作日志」**（工作流 ID 为 `worklog`，与命令等价）
- **URI**：`obsidian://rslatte-open?module=操作日志`（亦兼容 `module=时间轴`、`worklog` 等映射）

若统计子系统未初始化，侧栏会提示 **「统计功能未初始化」**。

---

## 数据从哪来

- 事件按 **空间** 分文件存放，由 `WorkEventReader` 读取。  
- 典型路径（`centralIndexDir` 取自 **统计/工作事件**相关设置中的中央索引目录，缺省时回退到插件主设置里的 **中央索引目录**）：

  `{centralIndexDir}/{spaceId}/.events/work-events-{YYYYMM}.jsonl`  

  若不存在，会尝试同结构下的 **`events/`**（不带点前缀）目录。

- 每月一个 **`.jsonl`** 文件，一行一条 JSON 事件。侧栏按你选的 **日期范围** 跨月读取并合并。

**前提**：插件需在设置中开启 **工作事件** 写入（见《用户手册》「工作事件与自动刷新」）；关闭后新事件不会落盘，本视图历史也会停在最后写入时刻。

## 筛选选项（默认收起）

点击 **「筛选选项」** 左侧 **▶ / ▼** 可展开或收起整块筛选区（默认 **收起**）。以下各小节也支持单独折叠。

### 日期范围

- 两个 **`type="date"`** 输入框：**开始日期**、**结束日期**（中间为「至」）。  
- 首次未选择时：**开始**默认为 **今天往前 7 天**，**结束**为 **今天**（本地日历 `YYYY-MM-DD`）。  
- 修改任一端后**立即**重新筛选并刷新列表。

### 空间筛选

- 列表来自设置 **`statsSettings.spaces`** 中 **已启用** 的空间（与插件主「空间」配置同步；详见设置页 **统计** 相关区块）。  
- 每个空间可配置 **名称**、**背景色**（用于时间轴卡片左侧色条，见下文）。  
- **全选**：勾选/取消当前列表中所有空间。  
- **新进空间**：**第一次**出现在该列表中的空间会 **默认勾选**；曾经出现过的空间若被你手动取消勾选，**不会**在下次打开时被自动勾回（避免覆盖「全不选」等刻意操作）。  
- 若未配置任何启用空间：时间轴区域提示在设置中添加空间配置。

### 模块筛选

- 模块 **kind** 列表来自 **`workEventSvc` 注册表**（各模块上报过的事件类型并集），排序后展示。  
- **`statsSettings.moduleEnabled`**：可按模块关闭勾选项（未配置的模块默认为 **启用**）；**关闭的模块不会出现在勾选列表中**。  
- **`statsSettings.moduleNames`**：可覆盖界面上的 **模块显示名称**（缺省用注册表里的 `module` 字段或内置中文映射，如任务、财务、输出等）。  
- **全选**只作用于当前 **已启用** 的模块键。

### 操作类型筛选

可多选以下 **action**（界面为中文标签）：

| 内部值 | 界面文案 |
|--------|----------|
| create | 创建 |
| update | 更新 |
| publish | 发布 |
| recall | 打回 |
| status | 状态变更 |
| delete | 删除 |
| archive | 归档 |
| cancelled | 取消 |
| done | 完成 |
| start | 开始 |
| recover | 恢复 |
| paused | 暂停 |
| continued | 继续 |

支持 **全选** / 单选；至少保留一类，否则提示 **「请至少选择一个操作类型」**。

### 事件来源筛选

| 内部值 | 界面文案 | 说明 |
|--------|----------|------|
| ui | 用户操作 | 界面操作产生；历史或未知来源也会归一为 `ui` |
| auto | 自动 | 插件自动逻辑产生 |
| reconcile | 数据同步 | 与对账/同步相关 |

至少选一个来源，否则提示 **「请至少选择一个事件来源」**。

### 其它门闩提示

- **未选任何空间**：「请至少选择一个空间」。  
- **未选任何模块**：「请至少选择一个模块」。  
- 筛选后无事件：「该时间段内没有符合筛选条件的事件」。  
- 加载异常：显示 **「加载失败：…」** 错误信息。

## 时间轴展示

- 事件先按 **日期** 分组，**日期从新到旧** 排列；标题形如 **`2026-04-05 (周六)`**（星期为中文）。  
- 同一天内按 **`ts` 时间字符串降序**（新的在上）。  
- 加载过程中会先显示 **「加载中…」**。

### 单条卡片上有什么

- **左侧圆点（轨道）**：根据 **模块 kind** 与 **action** 显示不同 **Emoji**（例如打卡 ✅、财务 💰、输出发布 📚、删除/归档 ⛔、任务状态 DONE ✅ / 进行中 ▶ 等）；具体映射见代码 `getModuleIcon`。  
- **主标题行**：优先显示事件的 **`summary`**；否则按 **`kind` + `ref`** 从字段里拼可读摘要（如任务正文、项目名称、财务分类+金额、快速记录文本等）。  
- **副行（元信息）**：**本地时间（时:分）** · **模块名** · **操作类型** · **空间名** · **事件来源**（与筛选区文案一致）。  
- **左侧竖条颜色**：由空间 **`backgroundColor`** 推导；若颜色过浅会改用主题强调色，避免看不清。鼠标悬停卡片可看到 **`空间：xxx`** 提示。

### 点击卡片

**单击整条事件**（非按钮区域）会按 **`kind` 打开对应侧栏**：

| kind | 打开目标 |
|------|----------|
| checkin | 打卡 |
| finance | 财务 |
| health | 健康 |
| task | 任务 |
| projecttask / project / milestone | 项目 |
| output | 输出 |
| contact | 联系人 |
| memo / schedule | 任务（提醒与日程合在任务侧栏） |
| capture | 快速记录 |
| 其它（如 file、sync 等） | **RSLatte 工作台（Hub）** |

跳转失败时会在控制台打出警告，界面不一定会弹 Notice。

## 设置里可配项（摘要）

在 **RSLatte 设置 → 统计/工作事件** 相关区域（实现上见 `renderStatsSettings.ts`）可影响本视图：

- **工作事件**总开关与写入路径习惯。  
- **`statsSettings.spaces`**：哪些空间出现在筛选列表、是否启用、显示名与颜色。  
- **`statsSettings.moduleEnabled` / `moduleNames` / `moduleColors`**：模块是否在操作日志中可选、显示名与颜色（颜色主要用于设置 UI；时间轴条主要用空间色）。
