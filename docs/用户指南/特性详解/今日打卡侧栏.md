# 今日打卡侧栏

## 概述

该侧栏视图类型为 **`rslatte-sidepanel`**（`RSLatteSidePanelView`），命令面板显示为 **「打开侧边栏：今日打卡」**。用于在 **当天** 一站式完成：

- **打卡**（勾选或弹窗备注）+ **近 30 天** 热力与计数；
- **财务**（按分类快速记账 + 本月支出条 + 本月/本年汇总）；
- **健康**（按设置启用的 9 项合并指标，一键录入/编辑今日值 + 30 天热图）；
- **今日日记**（可选：配置的日记子窗口列表 + 正文预览 + 跳转编辑）。

各分区可独立受 **模块开关** 控制；**财务饼图** 已迁至 **财务侧栏**，本侧栏不再展示饼图。

## 如何打开

- **命令**：`rslatte-open-sidepanel`（**「打开侧边栏：今日打卡」**）
- **Hub**：工作流条上的 **「今日打卡」** 按钮（与上者等价）
- **API / URI**：`activateRSLatteView(opts?)`；`obsidian://rslatte-open?module=今日打卡` 等

### 打开并滚动到指定分区

**`activateRSLatteView({ inspectSection })`** 会在侧栏显示后约 **180ms** 调用 **`scrollToInspectSection`**，滚动到带 **`data-rslatte-inspect`** 的分区：

| `inspectSection` | 对应区块 |
|------------------|----------|
| `checkin` | ✅ 打卡 |
| `finance` | 💰 财务 |
| `health` | ❤️ 健康 |
| `journal` | 📅 今日日记 |

（例如 Hub 日记芯片 **Alt+单击** 会打开本侧栏并滚动到 **日记** 区。）

---

## 顶栏「🔍 今日打卡」

- **🔄 统一刷新**  
  - 若 **打卡** 模块开启：对打卡执行 Pipeline **增量刷新**（不单独 `render`），并 **`recomputeCheckinContinuousDaysFromIndex`**。  
  - 若 **财务** 模块开启：对财务执行增量刷新。  
  - 最后 **整页 `render()` 一次**，Notice「已刷新」。  
  - **不**在按钮流程里直接刷新 **健康索引**；健康区依赖当前索引与打开侧栏时的数据。

---

## ✅ 打卡分区

- **标题行**：**✅ 打卡**；左侧可有 **DB 同步状态灯**（`appendDbSyncIndicator`）。  
- **➕**：**新增打卡项** → `AddCheckinItemModal`（在设置中维护的列表基础上增加）。  
- **展示样式**（`settings.checkinDisplayStyle`）：  
  - **`checklist`**：**复选框** + 名称 + 连续天数角标（若有）+ 难度图标槽 + 右侧 **近 30 天打卡天数**（按自然日去重，同日以最新记录为准，`is_delete` 视为未打）。点击行或勾选会 **直接切换今日打卡**（`performCheckinToggle`，**无弹窗**、备注为空）。  
  - **`buttons`（默认）**：**分类名按钮** → 打开 **`CheckinModal`** 可填备注；**连续天数**角标、**难度** emoji、右侧 **30 天热力**（颜色可取打卡项 `heatColor`）。  
- 数据：**今日完成状态** 来自 **`getOrCreateTodayState()`**；热力与 30 天计数来自 **打卡主索引** `getCheckinSnapshot(false)`（与完整打卡侧栏策略一致，今日汇总一般在 active 索引内）。

---

## 💰 财务分区

- **标题行**：**💰 财务** + DB 同步灯。  
- **➕**：**新增财务分类** → `AddFinanceCategoryModal`。  
- **分类列表**（仅 **启用** 的分类）：  
  - **左列**：按钮；**今日已有该分类有效流水** 时按钮为 **完成态**，并可显示 **笔数角标**；点击 → **`FinanceRecordModal`**。  
  - **右列**：支出类显示 **相对本月最大支出的比例色条**（条内金额）；收入类显示 **「收入条目」** 文案占位。  
  - 排序：**支出** 按 **本月支出额** 降序，**收入** 排在后面。  
- **统计两行**（纯索引计算）：**本月** 支出/收入、**本年** 支出/收入（基于 `recordDate` 与 `todayKey` 所在月、年）。  
- 索引来源：**财务主索引** `getFinanceSnapshot(false)`（与注释一致：与财务侧栏清单同源，不走路径不同的统计缓存分支）。

---

## ❤️ 健康分区

- **标题行**：**❤️ 健康**。  
- **📋**：**打开健康管理侧栏**（`activateHealthView`），用于完整清单与统计明细。  
- **明细区**：按 **`healthPanel`** 中 **UI 启用** 的合并项（`readHealthMetricsEnabledForUi`），对 **9 项** 顺序渲染（与健康管理侧栏一致）：体重、饮水、睡眠、饮食、腰围、血压、静息心率、血糖、月经周期。  
- 每一行：  
  - **按钮**：今日已有有效记录为 **done** 样式；点击 → 有则 **`EditHealthEntryModal`**，无则 **`HealthCardModal`**（`singleCanonicalMetric` + `lockAnchorToToday`）。  
  - **30 天热图**：该合并项在过去 30 个自然日是否 **有有效记录**（周/月卡按覆盖日展开，见 `buildHealthCanonicalHeatPresence`）。

---

## 📅 今日日记分区

- 受 **`settings.showJournalPanels`** 控制：为 **`false`** 时 **整块不显示**（默认展示，即 `!== false`）。  
- **➕**：**新增日志子窗口** → `AddJournalPanelModal`。  
- **`journalPanels`**：每个面板一行 — **按钮**（标签/标题）点击 → **`openTodayAtPanel(id)`** 打开编辑器定位；右侧 **预览** 异步读取当日文件片段（失败时仍可点按钮）。空内容显示「（空）」。

---

## 模块全关时的行为

- **`onOpen`**：若 **打卡、财务、健康** 三者 Pipeline **均未启用**，**不会**执行 `ensureTodayCheckinsInitialized` / `ensureTodayFinancesInitialized`，但仍会 **`render()`**。  
- **`render`**：仅 **打卡 / 财务 / 健康** 各自区块不渲染；**顶栏「🔍 今日打卡」与 🔄** 仍会画出；若 **今日日记** 仍开启，则仍可只用日记区。  
- 建议在设置中至少开启一类记录模块，否则侧栏信息量很少。

---

## 与 Hub / Today 的区别

| 入口 | 定位 |
|------|------|
| **今日打卡侧栏** | **当日** 打卡 / 财务快捷 / 健康快录 / 日记子窗口 |
| **今天（Today）** | 工作流页：**今日任务与执行**聚合，非本侧栏替代 |
| **Hub** | 空间总览与工作流跳转 |

勿与命令 **「打开侧边栏：今天」**（`rslatte-open-today-view`）混淆。

---

## 数据与刷新

- 打开侧栏时（模块开启前提下）会 **best-effort** 初始化今日打卡/财务（**默认不拉后端** `allowDb: false`），避免错误 API 配置刷红。  
- 写入成功后插件常 **`hydrateTodayFromRecordIndex`**、**`refreshSidePanel`**，使本侧栏与索引一致。  
- 若分类/按钮全灰或热力全空：检查 **日记路径**、执行对应模块 **刷新/重建索引**。
