# 工作台（Hub）

## 概述

**RSLatte 工作台**（`SpaceHubView`，视图类型 **`rslatte-hub`**）是 **多空间总览 + 工作流入口**：上方显示 **当前空间** 与 **刷新/切换** 操作；**工作流** 一排按钮打开各业务视图；中间为 **空间卡片网格**（每空间一行模块状态灯 + **日记**）；底部为 **空间告警**（按空间分页签、可展开模块详情、一键打开侧栏）。

打开时会 **先快速渲染**已有数据，再在后台 **`refresh()`** 并行刷新各空间统计，避免白屏过久。统计文件为各空间目录下的 **`space.json`**（路径由 `resolveSpaceStatsDir` 解析，随设置中的空间/索引配置变化）。

## 如何打开

- **命令**：`rslatte-hub-open`（**「打开侧边栏：RSLatte工作台」**）
- **功能区**：Ribbon **「打开 RSLatte工作台」**（图标 `layout-dashboard`）

---

## 页眉区域

| 元素 | 作用 |
|------|------|
| **标题 / 副标题** | 「RSLatte工作台」·「空间与工作流入口」 |
| **当前：{空间名}** | 展示插件当前激活空间（来自 `getCurrentSpaceId` + 空间配置名称） |
| **全部刷新** | 对 **所有已配置空间** 并行执行统计刷新：仅处理该空间 **已启用** 的 Hub 业务模块（见下「卡片模块顺序」），**强制**从队列拉数写入 `space.json`。刷新期间会抑制事件触发的防抖重绘，结束后 **立即** 再渲染一次。 |
| **切换空间** | 调用 **`openSpaceSwitcher()`**，与命令 **「RSLatte: Switch Space」** 同类 UI。 |

## 工作流按钮（实际排列顺序）

代码在 `WORKFLOW_VIEW_IDS` **之前** 额外插了两个按钮，因此 **从左到右** 为：

1. **日程日历** → `activateCalendarView()`  
2. **今日打卡** → `activateRSLatteView()`（侧栏 `rslatte-sidepanel`，含打卡/财务/健康/日记等）  
3. **快速记录**（`capture`）  
4. **今天**（`today`）  
5. **项目**（`projects` → 项目侧栏）  
6. **回顾**（`review`）  
7. **知识**（`knowledge` → 知识工作台视图）  
8. **操作日志**（`worklog` → 时间轴 `rslatte-stats-timeline`）

与常量注释「Hub 工作流条顺序」略有不同：**日程日历、今日打卡** 固定在 **最前**，其后才是 `WORKFLOW_VIEW_IDS` 六项。映射见 `WORKFLOW_TO_VIEW_TYPE`（`src/constants/viewTypes.ts`）。

## 空间卡片

### 无空间时

网格内提示 **「未配置任何空间」**，并引导到设置 **「空间管理」** 创建空间。

### 单张卡片结构

- **标题行**：空间名称；右侧  
  - **当前空间**：显示 **「当前」** 标签（不可点切换）；  
  - **其它空间**：**「切换到此空间」** → `switchSpace(id, { source: "hub" })`。  
- **当前空间卡片**会带高亮样式（`rslatte-hub-card-active`）。

### 模块条（芯片）

按固定顺序展示（`HUB_CARD_MODULE_KEYS`）：**任务 → 提醒 → 日程 → 打卡 → 财务 → 健康 → 项目 → 输出 → 联系人**，最后单独一块 **日记**。

- **是否出现**：某模块在该空间 **未启用且无统计条目** 时，芯片不渲染；否则显示（关闭模块时可能用占位 entry 展示「模块已关闭」类状态）。  
- **模块是否启用**：优先读该空间 **`settingsSnapshot.moduleEnabledV2`**（及旧版 `moduleEnabled` 映射），再回退全局 **`isPipelineModuleEnabled`** 等；且需 **`isPluginEnvInitModuleGateOpen`** 为真，否则 Hub 侧视为模块未开（与初始化门控一致）。

每块芯片上有两颗 **状态灯**：

| 灯 | 含义 |
|----|------|
| **内容灯**（前） | 根据该模块 **KPI** 由 `StatusCalculationService` 计算等级与 Emoji；模块未启用时为 🟠。悬停可看 **tooltip**（如今日任务、超期、打卡连续天、本月支出等）。 |
| **同步灯**（后） | 根据 `sync_status`（及 pending/failed）显示 DB 同步类状态 Emoji；悬停为同步说明。 |

**日记**芯片单独逻辑：依据 **今日日记文件是否存在** 与 **有效字数量**（去掉空白等后的「有意义字符」）；未创建时内容灯为 ⚪；有效字 **大于 10**（`HUB_JOURNAL_MEANINGFUL_OK_THRESHOLD`）视为达标。悬停可看具体有效字数或「未创建/无实质内容」等。

### 点击芯片

- **单击**：在下方 **「空间告警」** 区 **切换到该空间页签**，**只展开** 对应模块的 `<details>`，并 **滚动到可视区**（短暂高亮）。  
- **Alt + 单击**：在 **当前工作空间** 与卡片空间 **一致** 时，**直接打开对应侧栏**；若卡片是 **其它空间**，会 **Notice**「请先切换到该空间后再从工作台打开侧栏模块」。  
- **日记**芯片：**Alt + 单击** 打开 **今日打卡侧栏** 并带 **`inspectSection: "journal"`**（聚焦日记相关区域）。

## 空间告警（卡片下方）

- **标题**：「空间告警」。  
- **页签**：每个空间一个按钮；**当前空间**页签带额外样式。默认选中 **当前空间**（若列表中无当前空间则选第一个）。  
- **工具栏**：**全部展开** / **全部折叠** — 仅作用于 **当前选中页签** 下各模块的 `<details>`。  
- **每个模块一行**（`details`）：摘要行含 **模块中文名**、**「侧栏」按钮** 与 KPI/状态摘要；展开后列出 **告警行**（由 `buildHubFlatAlerts` 等汇总，含统计 KPI、财务/健康分析索引中的缺失类提示等）。  
- **「侧栏」**：与芯片 **Alt+单击** 相同规则 — **仅当已切换到该空间** 时才会打开对应模块侧栏；否则 Notice。  
  - 任务/提醒/日程 分别打开任务侧栏不同 **subTab**；日记 打开今日打卡侧栏日记区。

## 数据刷新与事件

- **`refresh()`**（页眉「全部刷新」与 `onOpen` 后台调用）：对每个空间调用 `SpaceStatsService.refreshSpaceStats`，模块列表为该空间 **Hub 上已启用** 的键。若某空间 **全部模块关闭**，则 **跳过** 刷新，避免无意义 IO 与事件循环。  
- **监听**：  
  - **`RSLATTE_EVENT_DB_SYNC_STATUS_CHANGED`**：仅刷新 **事件所指空间** 的对应模块统计，然后 **防抖 300ms** 再 `render`。  
  - **`RSLATTE_EVENT_SPACE_STATS_UPDATED`**：同样 **防抖渲染**。  
- 正在执行 **`refresh()`** 时，防抖渲染会 **直接 return**，避免闪烁与竞态。

首次进入某空间若 **`space.json` 缺失或缺模块块**，`loadStatsForHubSpace` 会 **自动补一次** 该空间启用模块的统计刷新。

## 与空间的关系

- **状态栏**、命令 **`rslatte-space-switch`**、Ribbon **切换空间** 与 Hub **切换到此空间** 互为补充。  
- 不同空间的 **路径、索引、`settingsSnapshot`** 独立，卡片与告警均 **按空间** 分开展示。  

详见 [空间与索引](空间与索引.md)。
