# 空间与索引

## 概述

RSLatte 的 **空间（Space）** 用于在同一 Vault 内隔离多套业务配置与 **按空间落盘的索引/队列/统计**；**中央索引目录**决定这些产物在库内的根位置。切换空间时，插件会把当前空间的设置写回快照、载入目标空间的快照，并刷新 API 作用域与内存缓存，避免读写串空间。

实现入口见 **`src/plugin/spaceManagement.ts`**、**`src/services/space/spaceContext.ts`**、**`src/types/space.ts`**。

## 空间标识与配置

- **空间 ID**：UUID 字符串，与后端 **`X-Space-Id`** 一致。默认空间固定为 **`00000000-0000-0000-0000-000000000000`**（常量 **`DEFAULT_SPACE_ID`**）。
- **展示名**：`RSLatteSpaceConfig.name`，用于状态栏、切换器等。
- **`spaceNumber`（可选）**：业务空间编号（如 1 表示默认空间），用于 V2 目录前缀等约定，与 UUID **独立**存储。
- **`settingsSnapshot`**：每个空间一份 **局部设置快照**（见下文「空间级设置」）。

空间列表保存在插件设置的 **`settings.spaces`**（按 id 索引的对象）；当前空间为 **`settings.currentSpaceId`**。

## 如何切换空间

- **命令**：`rslatte-space-switch`（打开全局空间切换模态框）。
- **Ribbon**：**layers** 图标 → 与上者相同，调用 **`openSpaceSwitcher()`**。
- **状态栏**：左侧 **`📁 空间名`** 项可点击，标题为「点击切换空间」；旁侧 **🔄** 为「刷新所有模块」（与单纯换空间不同）。

切换流程（**`switchSpace`**）要点：

1. 将 **当前** 全局设置中属于「按空间隔离」的字段 **提取** 为快照，写入 **`spaces[当前 id].settingsSnapshot`**。  
2. 设置 **`currentSpaceId`** 为目标 id。  
3. 用目标空间的 **`settingsSnapshot`** **覆盖** 全局设置中的空间级字段（**`applyPerSpaceSettings`**）。  
4. **`saveRaw`** 持久化（避免部分列表校验阻塞切换）。  
5. **`api.setSpaceId`** 对齐后端作用域；**`resetSpaceScopedCaches`** 清理内存缓存并重绑任务/输出/记录/项目等 store 的索引根目录。  
6. 触发工作区事件 **`rslatte:space-changed`**（**`RSLATTE_EVENT_SPACE_CHANGED`**）；约 50ms 后 **`refreshAllRSLatteViews`**，并 best-effort 触发项目 **`refreshAll`**。

若目标 id 不存在或当前已在该空间，会 Notice 提示，不会重复切换。

## 空间级设置 vs 全局设置

**`SPACE_SCOPED_SETTING_KEYS`**（**`src/services/space/spaceSettings.ts`**）定义哪些字段 **按空间隔离**，例如：模块开关（含 V2）、各 Panel 配置、打卡/财务分类、任务与日记路径、项目根目录与模板、Capture 目录、回顾模板开关等。

**刻意保持全局、不按空间切换** 的包括（注释中已写明）：**`centralIndexDir`**、**`vaultId`**、**`apiBaseUrl`**、WorkEvent 路径与开关、自动刷新索引开关与间隔、调试日志等。因此：**所有空间共用同一个「中央索引根」**，只是在该根下用 **`<spaceId>/`** 分子目录存放各空间数据。

日常开发理解：**`plugin.settings` 在运行时表示「当前空间生效的配置」**；持久化时每个空间的真实副本在 **`spaces[id].settingsSnapshot`**。

## 磁盘布局（中央索引根以下）

**中央根目录**由 **`resolveCentralRootDir`** 解析，优先级为：

1. **`settings.centralIndexDir`**（统一字段）  
2. 调用方传入的 legacy 候选路径（如部分模块旧版「索引目录」覆盖）  
3. **`settings.taskPanel.rslatteIndexDir`**  
4. 默认 **`00-System/.rslatte`**

在 **`centralRoot`** 下：

| 路径 | 含义 |
|------|------|
| **`spaces-index.json`** | 全库空间清单快照（**不按 space 分子文件**），由 **`SpacesIndexService`** 维护；位于中央根下。 |
| **`knowledge-index.json`** | 知识库轻量索引（**不按 space 分文件**），写在 **`resolveCentralRootDir`** 根下，与 `30-Knowledge` 扫描逻辑一致。 |
| **`<spaceId>/index/`** | 该空间的模块索引主目录（任务/提醒/日程、项目、输出、联系人、打卡与财务记录索引等，具体文件名因模块而异）。 |
| **`<spaceId>/queue/`** | 同步队列等（如项目队列子目录）。 |
| **`<spaceId>/stats/`** | 空间级统计聚合（如 **`space.json`** 与各模块 stats 片段，供工作台等读取）。 |
| **`<spaceId>/.events/`** | 事件类落盘（若模块启用）。 |

API：**`resolveSpaceBaseDir`** → `<centralRoot>/<spaceId>`；**`resolveSpaceIndexDir` / `resolveSpaceQueueDir` / `resolveSpaceStatsDir` / `resolveSpaceEventsDir`** 分别在基目录下追加 **`index`**、**`queue`**、**`stats`**、**`.events`**。部分模块在解析索引目录时会附带 **legacy 候选路径**（例如任务与项目各自设置里曾出现过的索引目录字段），与 **`spaceContext`** 注释一致。

### 常见索引文件名（按空间 `index/` 下，示例）

实际以各 **`indexStore` / Pipeline** 为准，常见包括：

- **任务侧栏**：`task-index.json`、`memo-index.json`、`schedule-index.json` 等。  
- **项目 / 输出 / 联系人**：`project-index.json`、`output-index.json`、`contacts-index.json`（及归档索引变体）。  
- **记录类**：`checkin-record-index.json`、`finance-record-index.json`、`health-record-index.json`、`rslatte-lists-index.json` 等。  
- **分析告警**（若生成）：如财务/健康侧栏提到的 `finance-analysis.alert-index.json`、`health-analysis.alert-index.json` 等路径（见对应模块文档）。

若 **SpaceStats** 发现某空间打卡/财务索引条目数异常，日志中会提示可能是 **其它空间数据误入**，应对 **该空间** 做索引重建而非随意删库根文件。

## 工作区事件与统计刷新

与空间相关的 Obsidian 工作区事件包括：

- **`rslatte:space-changed`**：当前空间已切换。  
- **`rslatte:space-stats-updated`**：某空间 **`stats`** 下聚合文件更新（如 **`SpaceStatsService.refreshSpaceStats`** 完成后）。  
- **`rslatte:db-sync-status-changed`**：DB 同步 pending/failed 等变化，Hub 等可随之刷新展示。

**`SpaceHubView`**（工作台）会监听统计与 DB 同步事件，并在手动刷新时对 **多个空间** 依次 **`refreshSpaceStats`**（仅处理该空间下已启用模块）。

## 与 Vault 业务目录的关系

**`ensureSpaceVaultDirectories` / `collectDirectoryPathsFromSpaceSnapshot`** 会根据 **空间快照** 中的日记路径、任务文件夹、Capture、项目根/归档、输出归档、联系人目录等 **在 Vault 内逐级建文件夹**。这与 **中央索引根** 不同：前者是「笔记与业务文件放哪」，后者是「RSLatte 机器可读索引放哪」。

V2 目录约定（如 **`10-Personal`**、**`30-Knowledge`**）需与 **当前空间的** 路径类设置、**插件初始化环境检查** 的检测结果一致或兼容；换空间后若路径指向另一套树，应重新确认检测与模板。

## 初始化门控

未完成 **插件初始化环境检查** 并执行 **「完成初始化（启用模块）」** 前，**模块管理**开关与业务侧栏行为以《用户手册》为准，可能与预期不一致。修改「文件与链接」、必选目录或模板后，应在检查向导中 **重新检测**。

## 刷新与重建

- 日常优先使用各侧栏或状态栏 **🔄 刷新**；**全量重建索引**耗时长，仅在索引损坏、换机恢复或文档明确建议时使用。  
- **不要随意删除正在使用的索引文件**；若误删，需了解对应模块如何触发重建，并注意 **当前 `spaceId`** 下的路径是否删对。  
- 多空间场景下，重建应针对 **当前空间** 或文档说明的范围，避免把其它空间的 `<spaceId>` 目录一并清空。
