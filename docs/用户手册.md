## 简介

RSLatte 是一个强大的 Obsidian 插件，旨在帮助你管理任务、项目、输出文档、财务记录、打卡记录、联系人等各类信息，并提供数据同步、统计分析和自动化归档等功能。

### 主要特性

- 📋 **任务管理**：支持任务追踪、状态管理、自动归档
- 📁 **项目管理**：管理项目、里程碑、任务清单
- 📄 **输出管理**：管理各类输出文档，支持模板创建
- 📣 **发布管理**：追踪文档发布记录和发布渠道
- 💰 **财务管理**：记录收支流水，支持分类统计
- ✅ **打卡管理**：记录日常打卡活动，支持热力图展示
- 👥 **联系人管理**：管理联系人信息，支持生日提醒
- 📝 **备忘管理**：记录各类备忘事项
- 📊 **统计分析**：提供时间轴、月度统计等数据分析功能
- 🏠 **工作台**：集中展示各类信息概览
- 🔄 **数据同步**：支持与后端数据库同步
- 📦 **自动归档**：自动归档过期数据

## 快速开始

### 安装插件

1. 打开 Obsidian 设置
2. 进入「第三方插件」
3. 点击「浏览」或「从文件安装」
4. 搜索并安装 RSLatte 插件
5. 启用插件

### 初始配置

1. **配置 API 地址**（如需要数据同步）
   - 打开设置 → RSLatte
   - 填写「API 基础地址」
   - 配置「Vault ID」（用于标识你的知识库）

2. **配置中央索引目录**
   - 默认：`95-Tasks/.rslatte`
   - 这是所有模块共享的索引存储位置

3. **启用需要的模块**
   - 在设置中启用你需要的功能模块
   - 各模块可独立开启/关闭

## 核心概念

### 空间（Space）

RSLatte 支持多空间管理，每个空间拥有独立的数据和配置：

- **默认空间**：首次使用时自动创建
- **切换空间**：通过 RSLatte Hub 侧边栏切换
- **空间隔离**：不同空间的数据互不干扰

### 模块（Module）

插件采用模块化设计，每个模块可独立启用：

- **任务模块**：管理任务和待办事项
- **项目模块**：管理项目和里程碑
- **输出模块**：管理输出文档
- **发布模块**：管理发布记录
- **打卡模块**：管理打卡记录
- **财务模块**：管理财务流水
- **联系人模块**：管理联系人
- **备忘模块**：管理备忘事项

### 索引（Index）

每个模块维护自己的索引文件，用于快速查询和同步：

- 索引存储在中央索引目录
- 支持自动刷新和手动刷新
- 支持索引归档

### 同步（Sync）

支持与后端数据库同步：

- 自动同步：定时同步数据
- 手动同步：通过刷新按钮触发
- 全量同步：重建索引并同步

## 功能模块

### 1. 任务管理

#### 功能特性

- ✅ 任务列表展示（进行中、已完成、已取消）
- 📅 任务日期管理（开始日期、计划日期、到期日期）
- 🏷️ 任务标签和分类
- 🔄 任务状态切换
- 📦 自动归档过期任务

#### 使用方法

1. **查看任务**
   - 打开「任务」侧边栏
   - 查看不同状态的任务列表

2. **新增任务**
   - 点击「➕ 新增任务」按钮
   - 填写任务信息（标题、日期、标签等）
   - 任务会自动写入日记

3. **编辑任务**
   - 点击任务项右侧的编辑按钮
   - 修改任务信息

4. **切换状态**
   - 点击任务状态按钮（进行中/已完成/已取消）
   - 状态会自动更新到日记

#### 配置说明

- **任务文件夹**：指定扫描任务的文件夹（默认：`01-Diary`）
- **包含标签**：只扫描包含指定标签的任务
- **排除标签**：排除包含指定标签的任务
- **自动归档**：启用后自动归档过期任务

---

### 2. 项目管理

#### 功能特性

- 📁 项目列表管理（进行中、已完成、已取消）
- 🎯 里程碑管理（支持多级里程碑）
- 📋 项目任务清单
- 📊 项目进度统计
- 📦 项目归档

#### 使用方法

1. **查看项目**
   - 打开「项目管理」侧边栏
   - 查看不同状态的项目列表

2. **新增项目**
   - 点击「➕ 新增项目」按钮
   - 填写项目信息
   - 系统会自动创建项目文件夹和任务清单

3. **管理里程碑**
   - 在项目卡片中点击「➕ 新增里程碑」
   - 或使用项目任务清单中的「➕」按钮新增子里程碑
   - 里程碑支持多级嵌套（最多 3 级）

4. **管理项目任务**
   - 在里程碑下点击「➕」按钮
   - 选择「📋 新增任务」或「🏁 新增里程碑」
   - 填写任务信息

5. **归档项目**
   - 完成的项目会自动归档
   - 也可手动归档

#### 配置说明

- **项目根目录**：项目文件存储位置（默认：`03-Projects`）
- **归档目录**：归档项目存储位置（默认：`03-Projects/_archived`）
- **任务清单模板**：项目任务清单模板路径
- **项目信息模板**：项目信息文档模板路径

---

### 3. 输出管理

#### 功能特性

- 📄 输出文档列表（进行中、已完成、已取消）
- 📝 模板创建文档
- 🏷️ 文档分类和领域
- 📦 文档归档（支持目录结构）

#### 使用方法

1. **查看输出文档**
   - 打开「输出管理」侧边栏
   - 查看不同状态的文档列表

2. **创建输出文档**
   - 点击「➕ 创建输出文档」按钮
   - 选择模板
   - 填写文档信息（标题、分类、领域等）
   - 系统会创建文档目录和文件

3. **管理文档状态**
   - 点击文档状态按钮切换状态
   - 状态会自动更新到文档 frontmatter

4. **归档文档**
   - 完成或取消的文档会自动归档
   - 归档时会移动整个文档目录

#### 配置说明

- **输出模板**：配置输出文档模板
- **归档目录**：归档文档存储位置
- **自动归档**：启用后自动归档过期文档

---

### 4. 发布管理

#### 功能特性

- 📣 发布记录管理
- 📊 发布统计（状态概览、通道统计、领域统计、时间趋势）
- 🌳 树形文件清单（分类 → 领域 → 文件）
- 🔍 多维度筛选

#### 使用方法

1. **查看发布文档**
   - 打开「发布管理」侧边栏
   - 查看发布统计和文件清单

2. **新增发布记录**
   - 在未发布的文档上点击「➕」按钮
   - 填写发布信息：
     - 发布通道（需在设置中配置）
     - 发布时间
     - 发布文档关联（可选）
     - 发布说明（可选）
   - 记录会以可读格式写入文档末尾的「发布信息清单」章节

3. **查看统计信息**
   - **发布状态概览**：查看已发布/未发布数量
   - **领域统计**：词云展示各领域的文档数量
   - **发布通道统计**：词云展示各通道的发布次数
   - **发布时间趋势**：按月份统计发布记录（可折叠）

4. **筛选文件**
   - 使用筛选器按文档分类、领域、发布类型筛选
   - 点击领域词云标签快速筛选

5. **树形文件清单**
   - 第一层：文档分类（可展开/折叠，显示文件数）
   - 第二层：领域（可展开/折叠，显示文件数）
   - 第三层：文件列表
   - 使用「全部展开」/「全部收起」按钮批量控制

#### 配置说明

- **文档目录**：指定扫描发布文档的目录（可多个）
- **发布通道选项**：配置可用的发布通道列表

---

### 5. 打卡管理

#### 功能特性

- ✅ 打卡记录管理
- 📅 打卡热力图（按日期展示）
- 📊 月度/年度统计
- 🔄 数据同步

#### 使用方法

1. **查看打卡记录**
   - 打开「打卡」侧边栏
   - 查看打卡热力图和记录列表

2. **新增打卡记录**
   - 点击「➕ 新增打卡」按钮
   - 选择日期、打卡项、备注
   - 如果该日期该打卡项已有记录，会提示不允许重复打卡

3. **打卡热力图**
   - 点击热力图中的日期可快速打卡/取消打卡
   - 切换到年度视图时，热力图按月份分行显示

4. **查看统计**
   - 切换到年度视图查看年度统计
   - 查看各打卡项的月度统计

#### 配置说明

- **打卡项配置**：在设置中配置打卡项（ID、名称、是否激活）
- **自动归档**：启用后自动归档过期记录
- **数据同步**：启用后与后端数据库同步

### 6. 财务管理

4. **筛选记录**

   - 按收入/支出类型筛选

## 设置配置

### 基础设置

#### API 配置

- **API 基础地址**：后端 API 地址（用于数据同步）
- **Vault ID**：知识库唯一标识

#### 空间设置

- **当前空间**：当前使用的空间 ID
- **空间列表**：管理所有空间

#### 中央索引目录

- 所有模块共享的索引存储位置
- 默认：`95-Tasks/.rslatte`

### 模块设置

每个模块都有独立的设置项：

#### 任务模块

- **任务文件夹**：扫描任务的文件夹列表
- **包含标签**：只扫描包含指定标签的任务
- **排除标签**：排除包含指定标签的任务
- **自动归档**：是否启用自动归档
- **归档阈值天数**：多少天后的任务自动归档
- **数据同步**：是否启用数据库同步

#### 项目模块

- **项目根目录**：项目文件存储位置
- **归档目录**：归档项目存储位置
- **模板路径**：各种模板文件路径
- **自动归档**：是否启用自动归档
- **数据同步**：是否启用数据库同步

#### 输出模块

- **输出模板**：配置输出文档模板
- **归档目录**：归档文档存储位置
- **自动归档**：是否启用自动归档

#### 发布模块

- **文档目录**：扫描发布文档的目录列表
- **发布通道选项**：可用的发布通道列表

#### 打卡模块

- **打卡项配置**：配置打卡项（ID、名称、是否激活）
- **自动归档**：是否启用自动归档
- **数据同步**：是否启用数据库同步

#### 财务模块

- **财务分类配置**：配置财务分类（ID、名称、类型、是否激活）
- **自动归档**：是否启用自动归档
- **数据同步**：是否启用数据库同步

#### 联系人模块

- **联系人目录**：联系人文件存储位置
- **联系人模板**：联系人文件模板路径
- **归档目录**：归档联系人存储位置
- **互动记录章节**：配置互动记录的章节标题

### 日记设置

#### 日记路径

- **日记目录**：日记文件存储位置（默认：`01-Diary`）
- **日记名称格式**：日记文件命名格式（默认：`YYYYMMDD`）
- **日记模板**：日记文件模板路径

#### 日志追加规则

配置各模块的日志追加规则：

- **模块**：选择模块
- **启用**：是否启用日志追加
- **H1 标题**：一级标题
- **H2 标题**：二级标题

#### 自动归档

- **归档阈值天数**：多少天后的日记自动归档
- **归档月份目录格式**：归档月份目录命名格式

### 统计设置

#### 工作事件流

- **启用工作事件**：是否启用工作事件记录
- **事件文件路径**：工作事件文件存储位置

#### 自动刷新

- **启用自动刷新索引**：是否启用自动刷新
- **刷新间隔（分钟）**：自动刷新间隔

## 使用技巧

### 1. 多空间管理

- 使用不同空间管理不同主题的内容
- 每个空间的数据和配置完全独立
- 通过 RSLatte Hub 快速切换空间

### 2. 模板使用

- 为项目、输出、联系人等配置模板
- 模板支持变量替换（如 `{{date}}`、`{{title}}`）
- 模板文件建议放在 `98-Templates` 目录

### 3. 标签和分类

- 使用标签对任务进行分类
- 使用文档分类和领域对输出文档进行分类
- 合理使用标签和分类可以更好地组织内容

### 4. 自动归档

- 启用自动归档可以保持知识库整洁
- 归档阈值可以根据需要调整
- 归档的数据仍可通过索引查询

### 5. 数据同步

- 启用数据同步后，数据会同步到后端数据库
- 支持双向同步（本地 → 数据库，数据库 → 本地）
- 同步状态会在侧边栏显示

### 6. 快捷键

- 使用命令面板快速打开各功能模块
- 配置 Obsidian 快捷键可以更快地访问功能

### 7. 筛选和搜索

- 各模块都支持筛选功能
- 使用筛选可以快速找到需要的内容
- 发布管理支持多维度筛选

### 8. 树形结构

- 发布管理的文件清单使用树形结构
- 合理使用展开/折叠可以更好地浏览内容
- 使用「全部展开」/「全部收起」批量控制

## 常见问题

### Q1: 如何备份数据？

A: RSLatte 的数据主要存储在：
- 索引文件：中央索引目录（`95-Tasks/.rslatte`）
- 源文件：各模块的源文件（日记、项目文件等）
- 建议定期备份整个 Obsidian 知识库

### Q2: 如何迁移到新设备？

A: 
1. 复制整个 Obsidian 知识库到新设备
2. 在新设备上安装 RSLatte 插件
3. 配置相同的 API 地址和 Vault ID（如需要同步）
4. 插件会自动识别索引和源文件

### Q3: 索引损坏怎么办？

A: 
1. 在对应模块的侧边栏点击「刷新」按钮
2. 如果刷新无效，可以删除索引文件后重新刷新
3. 索引文件位置：中央索引目录下的各模块索引文件

### Q4: 如何恢复已归档的数据？

A: 
1. 归档的数据仍在知识库中，只是移动到了归档目录
2. 可以通过文件管理器找到归档文件
3. 修改文件状态后，数据会重新出现在列表中

### Q5: 同步失败怎么办？

A: 
1. 检查 API 地址配置是否正确
2. 检查网络连接
3. 检查 Vault ID 是否正确
4. 查看控制台错误信息
5. 尝试手动刷新同步

### Q6: 如何自定义模板？

A: 
1. 在 `98-Templates` 目录创建模板文件
2. 在设置中配置模板路径
3. 模板支持变量替换，参考各模块的模板变量说明

### Q7: 如何批量导入数据？

A: 
1. 目前不支持批量导入
2. 可以通过 API 直接写入数据库
3. 然后刷新索引同步到本地

### Q8: 如何禁用某个模块？

A: 
1. 在设置中找到对应模块的设置
2. 取消勾选「启用」选项
3. 模块侧边栏会显示「模块未启动」提示

### Q9: 发布记录格式是什么？

A: 发布记录以可读的 Markdown 格式写入文档末尾：

```markdown
# 发布信息清单

- 📣 发布通道 | 发布时间 | 发布说明 | 📎 关联文档
  <!-- rslatte:publish:channel=...;date=...;relatedDoc=...;note=...;ts=... -->
```

### Q10: 如何查看详细的日志？

A: 
1. 在设置中启用「调试日志」
2. 打开开发者工具（Ctrl+Shift+I）
3. 查看控制台输出
